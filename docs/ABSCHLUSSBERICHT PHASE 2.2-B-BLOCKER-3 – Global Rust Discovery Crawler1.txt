ABSCHLUSSBERICHT: PHASE 2.2-B-BLOCKER-3 – Global Rust Discovery Crawler

Ich habe die Edge Function server-pulse-ingest wie gefordert auf einen zentralen "Global Discovery"-Ansatz umgerüstet und die Datenbank geprüft. Hier sind die Details:

Strategie & Root Cause
Root Cause (Alt): Die bisherige Nutzung von filter[search]=official/community/modded führte durch das harte Offset-Limit von 90 auf BattleMetrics dazu, dass niemals mehr als 100-180 Ergebnisse geladen werden konnten. Das ständige Resetten auf Seite 1 war die zwingende Folge.
Neue URL (Neu): https://api.battlemetrics.com/servers?filter[game]=rust&filter[status]=online&sort=-players&page[size]=100 Diese URL verzichtet auf das kaputte Search-Feld und nutzt sort=-players, was bei BattleMetrics sauberes Keyset-Pagination (page[key]) aktiviert. So können wir theoretisch Zehntausende Online-Server abgrasen.
Category global: Eingeführt. Die Function erkennt category: 'global' und baut dann die neue URL. Alle alten Kategorien (official, community, modded) sind zwecks Abwärtskompatibilität unberührt geblieben.
Parameter & Limits
page_size: Bleibt bei 100. Da wir nun Keyset-Pagination nutzen, ist dies kein Problem mehr.
maxPages pro Run: Bleibt bei 1. Ein Cron-Lauf = 1 Seite (100 Server).
max_page_window: Ich empfehle hier 50 (entspricht den Top 5000 Servern). Wir können es aber auch höher drehen, da wir jetzt wirklich tief scrollen können.
DB State & Cron (Vorbereitet, aber NICHT ausgeführt)
Ist eine DB Migration nötig? Nein. Ich habe information_schema.columns und pg_constraint geprüft. Es gibt keinen CHECK-Constraint auf die Spalte category. Wir können global einfach per SQL einfügen.
